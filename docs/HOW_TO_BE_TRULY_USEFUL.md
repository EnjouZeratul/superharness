## 一、核心公式

```
好用 = 低认知负担 × 高确定性 × 可预期结果

其中：
低认知负担 = 学习快 + 配置少 + 错误信息清晰
高确定性 = 行为可预测 + 无隐藏魔法 + 状态透明
可预期结果 = 5分钟见效 + 成功案例丰富 + 失败可恢复
```

### 1.1 与竞品对比

| 框架 | 认知负担 | 确定性 | 可预期性 | 综合评分 |
|------|---------|--------|---------|---------|
| **SmolAgents** | 极低(2步启动) | 高(代码即文档) | 中(功能简单) | 优秀 |
| **LangChain** | 高(概念多) | 中(抽象层多) | 高(生态成熟) | 良好 |
| **Continuum** | **需改进(当前9步)** | **待验证** | **待建立** | **起步阶段** |

## 二、用户放弃临界点

### 2.1 用户耐心曲线

```
时间/尝试次数        用户情绪          行为
────────────────────────────────────────────
0-2分钟/1-2次       充满期待          "试试看"
2-5分钟/3次         开始怀疑          "是不是我的问题？"
5-10分钟/3+失败     挫败感积累        "这东西太难用了"
>10分钟/放弃        彻底放弃          去找替代方案
```

### 2.2 放弃信号

| 信号 | 阈值 | 说明 |
|------|------|------|
| 连续失败次数 | ≥3次 | 用户开始怀疑自己能力 |
| 文档来回跳转 | ≥5次 | 信息架构混乱 |
| Google搜索次数 | ≥3次 | 文档覆盖不足 |
| 报错信息不可操作 | ≥1次 | 最后一根稻草 |

### 2.3 Continuum 当前风险

**Quick Start 步骤对比**：
```
SmolAgents (2步):
1. pip install smolagents
2. from smolagents import CodeAgent; agent.run("hello")

Continuum (当前9步):
1. pip install continuum
2. 创建配置文件
3. 配置API密钥
4. 定义Agent YAML
5. 注册工具
6. 创建Session
7. 启动Runtime
8. 执行任务
9. 查看结果
```

**结论**：认知负担过高，需精简至3-4步以内。

## 四、透明性与可控性设计

### 4.1 透明性三原则

```
原则1：无隐藏行为
- 所有操作可追溯
- 无后台自动调用
- 配置完全可见

原则2：状态可视化
- 当前执行到哪一步
- 下一步将要做什么
- 预计消耗多少资源

原则3：决策可解释
- 为什么选择这个工具
- 为什么终止循环
- 为什么需要用户介入
```

### 4.2 可控性三层级

| 层级 | 控制点 | 用户权限 |
|------|--------|----------|
| **L1 配置级** | YAML配置 | 完全控制 |
| **L2 运行时级** | 参数调整 | 动态修改 |
| **L3 干预级** | 手动介入 | 随时打断/恢复 |

### 4.3 可预测性保障

```python
# 不好的设计（不可预测）
agent.run(task)  # 黑盒，不知道内部发生了什么

# 好的设计（可预测）
with Agent(config) as agent:
    agent.set_max_iterations(10)      # 用户知道上限
    agent.set_cost_limit(1.0)         # 用户知道成本上限
    agent.set_timeout(300)            # 用户知道时间上限

    for step in agent.run(task):      # 用户看到每一步
        print(f"[{step.iteration}] {step.action}")
        print(f"  Tokens: {step.tokens_used}")
        if step.needs_confirmation:
            confirm(step.proposed_action)  # 用户可干预
```

## 五、错误恢复设计

### 5.1 三层恢复机制

```
┌─────────────────────────────────────────────────┐
│ Layer 3: 用户介入层                              │
│ - 手动Checkpoint恢复                             │
│ - 用户修改配置后继续                              │
│ - 人工审查并批准继续                              │
└─────────────────────────────────────────────────┘
           ▲ Layer 2失败后触发
┌─────────────────────────────────────────────────┐
│ Layer 2: 降级层                                  │
│ - 模型Fallback (GPT-4 → GPT-3.5 → Local)        │
│ - 功能降级 (复杂工具 → 简单工具)                   │
│ - 超时后继续 (保存状态，下次继续)                  │
└─────────────────────────────────────────────────┘
           ▲ Layer 1失败后触发
┌─────────────────────────────────────────────────┐
│ Layer 1: 自动重试层                              │
│ - 网络错误: 自动重试3次                           │
│ - Rate Limit: 指数退避                           │
│ - 临时故障: 等待后重试                            │
└─────────────────────────────────────────────────┘
```

### 5.2 错误信息设计原则

**不可接受的错误信息**：
```
Error: Something went wrong
Error: Agent failed
Error: Internal error
```

**可操作的错误信息**：
```
❌ 工具调用失败: search_files

原因: 目录路径不存在 "/project/src"
建议:
  1. 检查路径是否正确
  2. 使用绝对路径
  3. 确认工作目录: 当前在 /home/user

恢复选项:
  [R] 重试 (修改参数后)
  [S] 跳过此工具
  [C] 查看完整错误堆栈
  [Q] 退出并保存Checkpoint
```

## 六、SessionManager MVP设计

### 6.1 MVP核心功能（仅此一项）

```
MVP = 会话无缝延续

功能定义：
- Agent闪退后，无需重新发送消息即可继续
- 自动保存执行状态
- /continue指令恢复

不做：
- ❌ 多会话管理
- ❌ 会话切换
- ❌ 历史搜索
- ❌ 会话导出
```

### 6.2 Checkpoint最小设计

```python
@dataclass
class Checkpoint:
    """MVP最小Checkpoint"""
    # 必需字段
    session_id: str
    created_at: datetime
    messages: List[Dict]        # 完整消息历史
    iteration: int              # 当前轮次

    # 恢复必需
    pending_tool_call: Optional[Dict]  # 未完成的Tool Call

    # 成本追踪（透明性）
    tokens_used: int
    cost_estimate: float
```

### 6.3 存储简化

```
~/.continuum/
└── sessions/
    └── {session_id}/
        ├── latest.json         # 最新Checkpoint
        └── session_meta.json   # 会话元数据
```

**简化原则**：
- 只保留最新Checkpoint（不保留历史链）
- 文件名固定，无需排序
- 无符号链接，直接读写

### 6.4 恢复流程

```
用户执行: continuum continue

系统行为:
1. 检查 ~/.continuum/sessions/
2. 找到最新修改的session
3. 加载 latest.json
4. 验证 messages 完整性
5. 恢复到 AgentRuntime
6. 输出恢复信息
7. 继续执行

恢复信息示例:
────────────────────────────────────
📂 恢复会话: session_abc123
⏰ 最后保存: 2026-05-09 14:32:15
📍 恢复点: 第12轮，准备执行 tool_call
💰 已消耗: 15,420 tokens (~$0.03)
────────────────────────────────────
继续执行...
```

## 七、Quick Start 简化方案

### 7.1 目标：从9步精简至3步

**当前（9步）**：
```
1. pip install continuum
2. 创建配置文件
3. 配置API密钥
4. 定义Agent YAML
5. 注册工具
6. 创建Session
7. 启动Runtime
8. 执行任务
9. 查看结果
```

**目标（3步）**：
```
1. pip install continuum
2. export OPENAI_API_KEY=xxx
3. continuum run "帮我分析当前目录结构"
```

### 7.2 零配置设计

```python
# 内置默认配置，用户无需创建任何文件
DEFAULT_CONFIG = {
    "model": "gpt-4o-mini",        # 低成本默认
    "max_iterations": 10,
    "tools": ["read_file", "list_files", "write_file"],
    "checkpoint": {
        "enabled": True,
        "interval": 5              # 每5轮保存
    }
}
```

### 7.3 渐进式配置

```
Level 0: 零配置
         continuum run "task"
         └── 使用所有默认值

Level 1: 环境变量
         export SH_MODEL=gpt-4o
         export SH_MAX_ITER=20
         └── 覆盖默认值

Level 2: 配置文件
         continuum.yaml
         └── 完整自定义

Level 3: Python API
         from continuum import Agent
         └── 完全控制
```

### 7.4 一键示例

```bash
# 安装后立即体验
pip install continuum
export OPENAI_API_KEY=your_key

# 方式1: 命令行直接运行
continuum run "读取README.md并总结"

# 方式2: 交互模式
continuum chat
> 你好，我是Continuum Agent，请输入任务:
> 帮我分析当前项目的代码结构
[执行中...可以看到每一步]
[自动保存Checkpoint]

# 方式3: 恢复上次会话
continuum continue
```

## 十一、不做清单

### 11.1 MVP阶段坚决不做

```
❌ 多模型支持 (仅OpenAI)
❌ 复杂工具系统 (仅内置3个)

❌ 多会话管理 (仅单会话)

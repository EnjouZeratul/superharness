## 🚨 项目核心约定（必须遵守）

│                                                                     │
│  3. 任务定义必须精确                                                 │
│     ❌ "实现XX功能" （模糊）                                        │
│     ✅ "实现XX功能，达到YY产品级别，包含ZZ边界处理"                   │

│                                                                     │
│  5. 每个任务必须定义"完成标准"                                       │
│     ❌ 无明确验收标准                                               │

│                                                                     │

### 第一阶段: 产品级功能补齐 (P0 - 当前)

**目标**: 达到Claude Code/LangChain产品级体验

**对标分析**:

| 功能维度 | 当前状态 | Claude Code水平 | 差距 |
|----------|----------|-----------------|------|
| 工具链 | 无 | Bash/Read/Write/Edit/Grep/LSP完整 | 🔴 大 |
| Agent规划 | 无 | 任务分解、规划、自我纠错 | 🔴 大 |
| Git集成 | 无 | diff/commit/PR完整操作 | 🔴 大 |
| MCP协议 | 无 | 完整MCP客户端生态 | 🔴 大 |
| 文件操作 | 无 | 安全读写编辑、权限控制 | 🔴 大 |
| 错误恢复 | 基础 | 完整重试、降级、用户确认 | 🟡 中 |
| 流式输出 | 基础 | 完善的实时渲染 | 🟡 中 |

**Phase 1 任务清单**:

#### P1.1: 完整工具链实现 (T1 + T2)
- [ ] Bash工具 - 安全执行命令、超时控制、输出捕获
- [ ] Read工具 - 文件读取、大文件分页、编码处理
- [ ] Write工具 - 安全写入、备份、权限检查
- [ ] Edit工具 - 精确替换、多文件编辑
- [ ] Grep工具 - 内容搜索、正则支持、结果过滤
- [ ] Glob工具 - 文件模式匹配
- [ ] LSP工具 - 代码智能（定义跳转、引用查找）

#### P1.2: Agent智能增强 (T1)
- [ ] 任务分解 - 复杂任务自动拆解为步骤
- [ ] 执行规划 - 生成执行计划，可见可控
- [ ] 自我纠错 - 检测错误并自动修复
- [ ] 进度追踪 - 实时显示任务进度

#### P1.3: Git深度集成 (T2)
- [ ] git status/diff显示
- [ ] git commit集成（消息生成、变更确认）
- [ ] git branch管理
- [ ] PR创建和管理

#### P1.4: MCP协议支持 (T2)
- [ ] MCP客户端实现
- [ ] MCP服务器连接
- [ ] MCP工具调用

#### P1.5: 文件系统安全操作 (T1)
- [ ] 路径验证（禁止访问外部目录）
- [ ] 权限检查
- [ ] 操作审计日志
- [ ] 变更预览和确认

#### P1.6: 错误恢复完善 (T1 + T2)
- [ ] 三层重试机制（自动→降级→用户介入）
- [ ] 会话恢复（任意中断点可恢复）
- [ ] 用户确认机制（高风险操作）

**完成标准**:
```
✅ 真实用户能完成完整开发任务（如：修复bug、添加功能）
✅ 工具链功能完整，操作安全可靠
✅ Agent能自主规划执行复杂任务
✅ Git操作完整可用
✅ 错误恢复机制完善，不会因错误崩溃
```

## 八、历史教训记录

### 2026-05-12 Phase 0反思

**问题**: SDK/TUI交付的是骨架/Demo，不是完整产品

**根本原因**:
1. 任务定义不精确："实现XX功能"未明确"达到YY产品级别"
2. 验收标准模糊：只看测试通过，没看真实使用
3. 缺少对标基准：没有明确定义"持平Claude Code"的具体指标

**改进措施**:
1. 每个任务必须定义"完成标准"，包含功能点、边界处理、错误恢复
2. 验收看真实使用流程，不看测试通过
3. 对标Claude Code/LangChain，明确差距和补齐计划
